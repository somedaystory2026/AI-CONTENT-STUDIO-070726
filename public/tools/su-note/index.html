    SU-Note (수-노트) 는 Suno AI 음악 생성에 최적화된 프롬프트 매니저 입니다. Excel 형식의 Suno AI 한국어 종합 가이드 v8 을 기반으로 648 개 검증된 고유 태그, 867 개 스타일 템플릿 (261 큐레이션 + 202 아티스트 × 3 변형 = 606 시그니처), 200 가지 댄스 서브장르를 한 화면에서 검색·조합·복사할 수 있도록 정리했습니다.

    🧹 자동 정제 — Suno 영문 토큰화 보장

    Suno 의 Style 필드는 영문 음악 용어만 인식합니다. 한국어·아티스트명·언어 지시가 섞이면 Suno 의 토큰화가 깨지므로, SU-Note 는 모든 Style 칩 진입 경로 (프리셋 로드 · AI 출력 · 수동 입력 · 팔레트 · 댄싱 머신) 에서 자동으로 정제합니다.

    ① 한국어 음악 용어 → 영문 치환

    • 트로트 → Trot · 발라드 → ballad · 동요 → children song · 통기타 → acoustic guitar folk
    • 한국 가요·한국 힙합·한국 인디·시티팝 → Korean pop · Korean hip-hop · Korean indie · city pop
    • 걸그룹·보이그룹·아이돌 → girl group · boy group · idol
    • 매핑되지 않은 한글은 모두 제거 (잔여 한글 0건 보장)

    ② 아티스트·프로듀서·밴드멤버 이름 → 역할 라벨

    • 대표곡 변형 606 곡 + Excel 가이드 템플릿 + 댄스 200 스타일에서 모든 실명 자동 제거
    • 예시 치환:
      • Quincy Jones / Phil Spector / Mutt Lange → the producer
      • Nile Rodgers / Slash / Jimmy Page → funk guitarist / the lead guitarist
      • McCartney / Lennon / Mercury / Plant / Ozzy → the lead vocalist
      • Bonham / Moon / Ulrich → the drummer
      • Bradley Cooper / Gaga / Adele / Beyoncé → the male/female vocalist
      • Daft Punk / Tiësto / Noisia / Skrillex → French house duo / trance producer / 등
      • RZA · GZA · Snoop · Cardi B · Megan → the rapper / featured rapper
    • 총 250+ 패턴 적용 (250+ 인명 → 역할 라벨, 소유격·접미 패턴까지 정리)
    • 아티스트 카드의 라벨 (예: BLACKPINK · BTS · Adele) 은 그대로 유지 — 어디까지나 Style 텍스트 안에 들어가는 인명만 제거

    ③ 언어 지시 자동 제거

    • 가사 언어는 Lyrics 자체 가 결정합니다. Style 에 들어가는 언어 지시는 노이즈이므로 모두 제거:
    • vocals in Korean · in English · with Korean · Korean lyrics · English lyrics · bilingual · codeswitching · Korean pronunciation → 모두 빈 문자열
    • 가사 한·영 비율은 우측 제어 패널의 가사 언어 페이더 로만 조정 → AI 가 가사 자체를 해당 언어로 작성

    ④ 가사 섹션 빈 줄 자동 삽입

    • AI 생성 · 프리셋 로드 · 팔레트 클릭 · 기본 골격 삽입 모든 경로에서 [Section] 헤더 앞에 빈 줄 1줄 자동 보장
    • 3 줄 이상 연속 빈 줄은 1 줄로 정리 (멱등 정규화)
    • 가사가 시각적으로 블록 단위로 구분되어 가독성·편집 편의 ↑

    ⑤ 중복 era / genre 꼬리 태그 자동 제거

    • AI 가 자주 만들던 중복 패턴 — 앞에 2010s K-pop 박고 꼬리에 2015-2019 Korean idol pop · early-2020s K-pop production · global K-pop dominance era 같은 재진술 — 을 자동 dedup
    • 중복 시대 토큰 (decade · year · range) 은 첫 것만 유지
    • 중복 장르 lineage (K-pop / J-pop / Korean idol group 등 동의 그룹) 도 첫 것만 유지
    • Filler 꼬리 패턴 (X-era sound · X-era production · dominance era · global X dominance 등) 통째로 drop
    • AI 생성·검색 결과·프리셋 로드 모든 경로에서 적용 → 200자 token budget 효율 사용

    ⑥ Suno 적용도 기반 칩 자동 정렬

    • 로드된 칩은 카탈로그의 reflect 데이터 와 매칭돼 자동 stable 정렬: 🟢 (잘 됨) → 🟡 (가끔만) → ⚪ (데이터 없음) → 🔴 (잘 안 됨)
    • 200자 token 한도에서 잘릴 때 핵심 anchor 태그가 살아남도록 우선순위 강제
    • 같은 적용도 안에선 추가된 순서 보존 (stable sort)
    • 각 칩 좌측 ● 도트 색상으로 적용도 시각화 → 마우스 오버 시 한국어 설명
    • 🔴 칩은 살짝 흐려져 (opacity 0.7) 우선순위가 낮음을 표시. 사용자가 의도적으로 두면 그대로 유지

    ★ 모든 정제는 실시간 런타임 으로 동작하므로 향후 Excel 가이드가 갱신되어 인명·한국어가 새로 들어와도 자동 처리됩니다.

    ❓ 자주 묻는 질문 (FAQ)

    Style 채우기 체크가 꺼져있어요
    프리셋을 로드하면 자동으로 잠깁니다 (Excel 원본 보호). 🔒 배지 클릭으로 해제하면 AI 가 Style 을 새로 작성합니다.
    아티스트명이 Suno 로 전달되나요?
    아니요. 아티스트명은 사용자가 스타일을 찾기 쉽도록 돕는 분류 라벨 일 뿐이며, Suno 에 가는 프롬프트엔 포함되지 않습니다. 4 곳에 명시되어 있습니다.
    가사가 한글로 안 나와요
    제어 패널의 가사 언어 페이더 가 한글 100% (좌측 끝) 인지 확인하세요. AI 가 페이더 값을 그대로 따릅니다.
    같은 칩이 두 번 들어가요
    대·소문자 무관 자동 중복 제거됩니다. 보인다면 Ctrl+Shift+R 강력 새로고침.
    Style 에 한글 (트로트 · 발라드) 이 보여요
    모든 한국어 음악 용어가 자동으로 영문 치환됩니다 (트로트 → Trot, 발라드 → ballad). 만약 보인다면 강력 새로고침 후 다시 시도하세요.
    Style 에 아티스트 이름이 보여요 (예: McCartney · Bradley Cooper)
    250+ 인명 패턴이 자동으로 역할 라벨 (the lead vocalist, the producer 등) 으로 치환됩니다. 갓 추가된 인명은 다음 빌드에서 잡힙니다.
    Style 에 "in English / in Korean" 같은 표현이 들어가요
    언어 지시는 Style 에서 자동 제거됩니다. 가사 언어는 우측 제어 패널의 가사 언어 페이더로만 결정됩니다.
    시작점 패널이 안 보여요
    시작점 패널은 Style 칩이 비어 있고 프리셋이 로드되지 않은 상태에서만 자동 표시됩니다. 칩이 1 개라도 있으면 (수동 추가·다른 탭에서 전송 등) 자동으로 숨겨져요. 다시 보려면 초기화 버튼으로 빌더를 리셋. 또는 좌측 프리셋 · 댄싱머신 · 검색 탭에서 스타일을 골라 빌더로 전송해도 됩니다.
    칩 순서가 자동으로 바뀌어요
    의도된 동작입니다. 칩은 Suno 적용도 (reflect) 기준으로 자동 정렬돼요 — 🟢 → 🟡 → ⚪ → 🔴. Suno 의 200자 token 한도에서 잘릴 때 잘 적용되는 태그가 살아남도록 핵심을 앞쪽에 모읍니다. 칩 좌측 도트 색상이 적용도를 표시 (마우스 오버 시 한국어 설명). 같은 적용도 안에선 추가된 순서가 유지됩니다 (stable sort).
    칩에 빨간 도트가 나오는 태그는 지워야 하나요?
    꼭 그럴 필요는 없습니다. 🔴 은 "Suno 가 잘 반영 안 함" 이지 "잘못된 태그" 가 아니에요. 의도적으로 무드·스토리 신호용으로 두고 싶으면 그대로 유지. 200자 한도에 가까울 때 우선순위 정렬로 뒤에 밀려있어 자동으로 잘릴 수도 있고, × 버튼으로 직접 제거해도 됩니다.
    "Style에 키워드 추가" 버튼이 사라졌어요
    "빌더에 로드 · 제어 옵션 머지" 버튼에 통합됐습니다. 콜앤리스폰스 또는 화음·코러스가 켜져 있으면 선택한 보컬 편곡 스타일의 키워드가 로드 시 자동 추가됩니다.
    "빌더에 로드" 버튼이 두 가지 모드로 동작한다는데?
    네 — ① 시작점 패널에서 프리셋을 선택한 상태면 프리셋 Style + 곡 구조 + 제어 옵션 을 모두 적용. ② 검색·댄싱머신·태그 탭에서 가져온 칩만 있는 상태면 제어 옵션 (콜앤리스폰스·화음·코러스·보컬 편곡 스타일) 만 기존 칩에 머지. 어느 쪽이든 한 번 클릭으로 끝.
    탭 전환할 때 빌더·앨범 작업 상태가 유지되나요?
    이젠 초기화되지 않습니다. 사이드바 탭은 순수 네비게이션이라 빌더의 칩·제목·가사·AI 프롬프트, 검색 결과, 댄싱머신 picks, 앨범 12곡 진행 상태가 탭을 옮겨도 그대로 유지됩니다. 특히 앨범 가사 일괄 생성처럼 시간이 걸리는 작업 중에 다른 탭을 봐도 안전합니다. 비우고 싶을 땐 각 탭의 초기화 버튼을 직접 누르세요.
    해외 아티스트를 한글로 검색해도 되나요?
    네 — AI 가 한글 음역을 자동 인식합니다. 예: 콜드플레이 → Coldplay · 비틀즈 → The Beatles · 다프트펑크 → Daft Punk · 빌리아일리시 → Billie Eilish · 요네즈켄시 → Kenshi Yonezu. 노래 제목도 동일: 보헤미안랩소디, 옐로우 등. 한국 아티스트 (방탄소년단 · 아이유 등) 는 그대로 한글 검색.
    가사 섹션 사이에 빈 줄이 자동으로 들어가요
    가독성을 위한 의도된 동작입니다. [Verse 1] / [Chorus] 등 모든 섹션 헤더 앞에 빈 줄이 자동 보장됩니다. 직접 편집해도 다음 AI 생성 / 프리셋 로드 시 다시 정규화됩니다.
    200 댄스 스타일에 원하는 게 없어요
    좌측에서 가장 가까운 시드 스타일을 고른 뒤 우측 태그 조합기에서 카테고리별 태그를 골라 직접 조합하세요. 8 카테고리 × 210 태그 = 충분한 변주가 가능합니다.
    앨범 12곡 가사 일괄 생성이 너무 느려요
    곡당 약 5~15 초 × 12 = 1~3 분 소요. AI 모델별로 응답 속도 다름. 진행 상황은 상태 바에 표시됩니다.
    AI 가 만든 가사가 제 의도와 달라요
    AI 프롬프트 입력칸을 더 구체적으로 — 시대·무드·이야기·핵심 단어를 명시. 페이더로 비유 강도·언어·랩 비율도 조정.
    오프라인에서 동작하나요?
    UI 는 정적 SPA 라 첫 로딩만 인터넷 필요. 단 AI 생성은 항상 외부 API 호출이 필요합니다.
    가사 없는 연주곡을 만들고 싶어요
    AI 패널의 🎺 연주곡 체크박스 ON → Style 칩에 instrumental 자동 삽입, Lyrics textarea 비워짐, AI 는 sung 라인 없이 섹션 로드맵만 작성. 7-9 섹션 × 80-120 마디로 3-4분 트랙 보장. Suno Custom Mode 에 그대로 붙여넣으면 됩니다.
    제목만 다른 언어로 만들고 싶어요 (예: 한국어 가사 + 영문 제목)
    AI 패널의 제목 언어 드롭다운에서 English 등 선택. Auto 면 가사 언어 페이더를 따르고, 나머지는 가사와 무관하게 제목만 강제 언어로 작성합니다. 연주곡 모드에서도 동작.
    분할 모드에서 SU-Note 가 Suno 입력란에 직접 붙여넣을 수 있나요?
    불가능합니다. 브라우저 보안 (Same-Origin Policy) 상 SU-Note 와 Suno.com 은 서로 다른 출처라 DOM 접근이 차단됩니다. 📋 Style 복사 · 📋 Lyrics 복사 → Suno 창 클릭 → Ctrl+V 가 유일한 방법.
    분할 모드 ON / OFF 시 워크플로우 차트가 사라져요
    사라진 게 아니라 사이드바 안 컴팩트 스트립 으로 자동 이동합니다. 일반 모드로 돌리면 빌더 본문 상단 원래 자리로 복귀합니다.
    댄싱머신 / 저장된 프리셋을 빌더로 보냈더니 AI 가 엉뚱한 스타일로 가사를 써요
    최근 패치로 수정됐습니다. 이전엔 빌더 칩만 갈아끼우고 AI 가 참조하는 state.selectedArtist 가 이전 프리셋 상태로 남아 있어 그 아티스트 스타일이 강제됐어요. 지금은 댄스 전송 · 저장 프리셋 복원 · 앨범 트랙 송신 모든 흐름에서 잔여 컨텍스트가 자동 초기화됩니다.
    Suno 가 연주곡인데 90초로 잘려 나와요
    Suno 는 Lyrics 필드의 글자 수와 섹션 밀도를 곡 길이 신호로 사용합니다. 연주곡 모드는 AI 에게 7-9 섹션 + 모든 브래킷에 명시적 마디 수 (8/16/24 bars) + Style 에 "extended arrangement" 길이 힌트를 강제하도록 설계됐어요. 그래도 짧다면 같은 프롬프트로 한 번 더 재생성 (Suno 자체의 변동성 가능).
    제 데이터는 어디 보관되나요?
    모든 프리셋·앨범·API 키·테마·워크플로우 상태는 브라우저 localStorage 에만 보관됩니다. 외부 서버 전송 없음.
    모바일에서도 쓸 수 있나요?
    반응형 디자인 적용. 1100px 이하에서 사이드바·페이더·태그 그리드가 세로로 재배치됩니다.
    저장한 프리셋·앨범이 갑자기 사라졌어요
    브라우저의 "쿠키 및 사이트 데이터 삭제" 또는 시크릿 모드 종료로 localStorage 가 초기화된 경우입니다. 📥 가져오기 로 Downloads/suno_prompt_manager/ 에 백업해 둔 JSON 파일을 다시 불러오세요. 백업이 없다면 복구 불가능. ⚠️ 정기적인 📤 JSON 내보내기를 권장합니다.
    다른 PC·브라우저로 작업을 옮기고 싶어요
    현재 기기에서 📤 JSON 내보내기로 받은 파일을 새 기기로 옮긴 뒤, 동일 도메인 (또는 동일 로컬 서버) 에서 📥 가져오기로 로드. 같은 도메인이 아니면 localStorage 가 격리되어 자동 공유되지 않습니다.
